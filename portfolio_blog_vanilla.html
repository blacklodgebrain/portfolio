Vanilla JavaScript vs. Next.js React in Building Todo Apps

In the world of web development, there are numerous frameworks and libraries available to streamline the process of creating dynamic and interactive web applications. Two popular options are Vanilla JavaScript and React, a JavaScript library for building user interfaces. Within the React ecosystem, Next.js stands out as a powerful framework that enhances React development by providing features like server-side rendering and routing out of the box. In this blog post, we'll delve into the experience of building a simple Todo application using both Vanilla JavaScript and Next.js React, highlighting the differences and advantages of each approach.

                        <!DOCTYPE html>
                        <html lang="en">
                        <head>
                            <meta charset="UTF-8">
                            <meta name="viewport" content="width=device-width, initial-scale=1.0">
                            <title>Vanilla JS Todo App</title>
                            <link rel="stylesheet" href="styles.css">
                        </head>
                        <body>
                            <div class="todo-container">
                                <input type="text" id="todo-input" placeholder="Add a new todo">
                                <button id="add-btn">Add</button>
                                <ul id="todo-list"></ul>
                            </div>
                            <script src="app.js"></script>
                        </body>
                        </html>

In this Vanilla JavaScript implementation, we manually handle user input, create elements dynamically, and update the DOM accordingly. While this approach offers simplicity and flexibility, it can become cumbersome and error-prone as the application grows in complexity.

                        # Create a new Next.js project
                        npx create-next-app nextjs-todo

                        // pages/index.js
                        import { useState } from 'react';

                        export default function Home() {
                            const [todos, setTodos] = useState([]);
                            const [todoText, setTodoText] = useState('');

                            const handleAddTodo = () => {
                                if (todoText.trim() !== '') {
                                    setTodos([...todos, todoText]);
                                    setTodoText('');
                                }
                            };

                            return (
                                <div className="todo-container">
                                    <input  
                                        type="text"  
                                        placeholder="Add a new todo" 
                                        value={todoText} 
                                        onChange={(e) => setTodoText(e.target.value)} 
                                    />
                                    <button onClick={handleAddTodo}>Add</button>
                                    <ul>
                                        {todos.map((todo, index) => (
                                            <li key={index}>{todo}</li>
                                        ))}
                                    </ul>
                                </div>
                            );
                        }

In this Next.js React implementation, we leverage React's component-based architecture and state management through hooks like useState. The code is more declarative and easier to understand, promoting better organization and maintainability. Additionally, Next.js handles routing and server-side rendering seamlessly, improving performance and SEO.

While both Vanilla JavaScript and Next.js React offer ways to build Todo applications, the latter provides a more structured and efficient development experience. With Next.js, developers can leverage the power of React along with additional features like server-side rendering and routing out of the box, resulting in faster development cycles and better performance. However, the choice between Vanilla JavaScript and Next.js React ultimately depends on the specific requirements and preferences of the project at hand.
